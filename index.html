Our solution

Resolve your environment before the session opens.

Perseus is a pre-processor. You write directives once in a markdown source file. At render time Perseus resolves them (git state, service health, test status) into a plain file your assistant reads. One install, one command. The assistant opens knowing what is running.

context.md.perseus
# source: you write this once
Branch: @git.branch
Services: @services.health
Tests: @test.status

# rendered: the assistant reads this
Branch: feat/payments (4 ahead)
Services: api up · db up
Tests: 1185 passing
