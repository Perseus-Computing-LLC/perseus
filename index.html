§ How It Works

The assistant never sees a directive. It reads a file that was already true.

Directives resolve to live facts before the agent opens the file. No runtime queries. No orientation turns. Just a plain markdown file with current state.

Without Perseus — 4 orientation turns
# session opens, CLAUDE.md last edited 11 days ago
you  deploy the api service to staging.
ai   running `make deploy-staging`...
ai   Makefile target removed in #482.
     Checking the current scripts.
ai   Which branch are you on? Is the
     helm chart still v0.3? What's the
     cluster name?
# 4 orientation turns. 11k tokens. 0 progress.
With Perseus — 1 turn
# CLAUDE.md rendered 38s ago by perseus render
you  deploy the api service to staging.
ai   You're on feat/billing-v2, helm
     chart v0.4.1, staging =
     eu-west-2/stg-a.
ai   Running `./scripts/deploy.sh staging
     api`.
ai   ✓ Image pushed. ✓ Helm upgrade applied.
# 1 turn. Branch, chart, cluster: all true.
§ Platform

Everything an agent needs, before it asks

Perseus provides a unified pipeline: render context, recover state, rank actions.

⟳

Live Renderer

Resolves @query, @read, @waypoint, and @services directives into plain markdown. Tiered progressive disclosure with <31ms render time.

◎

Session Waypoints

Save checkpoints at natural pauses. Recover state across sessions without re-orientation. Diff waypoints to see what changed.

◆

Pythia Tool Oracle

Surfaces the highest-utility skill or tool for a given task. No extra model required — operates in the same context window.

⬡

Progressive Disclosure

Tier 1, 2, and 3 context levels. Give agents exactly what they need — no more, no less — while keeping full auditability.
