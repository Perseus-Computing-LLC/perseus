      // The determinism wedge
      
Probabilistic memory guesses. Perseus returns the same answer, every time.

      
Cloud memory extracts facts with an LLM into a hosted vector store, so the same query drifts. Perseus Vault is FTS5, SQL, and full bi-temporal history: replayable at any point in time, offline, and audited.

      // How it works
      
Two engines resolve the two things assistants forget: live context and durable memory.

      
Perseus resolves your environment into ready facts before the session opens. Perseus Vault writes verified decisions into deterministic storage and recalls them on demand. Both run entirely on your hardware.

      

        

          

            

              

                
              

              
Perseus
Live context engine

            

            context
          

          
Write directives once. Perseus resolves git, services, and tests into verified facts, then writes them into the file the assistant reads (AGENTS.md / CLAUDE.md).

          
$ perseus render
✓ 4 directives in 31ms
✓ wrote CLAUDE.md # 76% smaller
