# Changelog

All notable changes to Perseus.

This project follows the alpha-then-semver convention:
during the alpha (pre-`1.0.0`) phase, minor bumps may include breaking changes
that are documented in the release notes.

Each entry maps a release to the task IDs that shipped in it. The
single-file `perseus.py` runtime is the only required artifact; everything
else (installer, docs) is generated by `scripts/release.sh`.

## [1.0.6] тАФ UNRELEASED

Critical security + correctness hotfix bundle. See GitHub milestone
[v1.0.6](https://github.com/tcconnally/perseus/milestone/1).

### ЁЯФТ Security

- **#137** тАФ `@query` audit log no longer leaks secrets. Pre-1.0.6, calls
  like `@query "curl -H 'Authorization: Bearer ghp_тАж'"` rendered correctly
  redacted output but persisted the raw bearer token to
  `~/.perseus/audit_log.jsonl` (via the `command` field), and leaked the
  same secret in `@query` error/timeout/no-output messages back into
  render output.
  - `audit_event` (audit.py) now passes every user-supplied field value
    through `redact_text` before writing. Structural fields (`directive`,
    `exit_code`, `duration_ms`, `pid`, etc.) are exempt via an explicit
    allowlist (`_AUDIT_NEVER_REDACT_KEYS`).
  - `resolve_query` (directives/query.py) redacts `cmd`, `stderr`, and
    exception messages before interpolating them into the error/timeout/
    no-output strings.
  - New config knob `audit.redact_fields` (default `true`); set `false`
    to opt out for forensic mode where the audit log is itself the
    secured artifact.
  - Nested structures (dicts, lists) are walked recursively.

### ЁЯРЫ Bug Fixes (other v1.0.6 items, tracked in milestone)
- #128, #129, #130, #131, #135, #136, #138, #139, #140, #141, #142

### ЁЯУж Migration Notes
- No config breaking changes. The new `audit.redact_fields` default of
  `true` is strictly more secure than pre-1.0.6 behavior.

## [1.0.4] тАФ 2026-05-25

**Phase 24 тАФ Extensibility Architecture (Hephaestus):**

- **task-65** тАФ Plugin directive system: auto-discovered Python plugins under `~/.perseus/plugins/`. Each module exports a `REGISTER` dict of `DirectiveSpec` entries. Plugin errors are warnings, not fatal.
- **task-66** тАФ Directive macros: `@macro name ... @endmacro` blocks in context documents or `.perseus/macros.md`. Pre-processing pass expands invocations before the resolver loop.
- **task-67** тАФ Render pipeline hooks: lifecycle callbacks (`on_render_start`, `on_directive_resolved`, `on_cache_hit/miss`, `on_render_complete`, `on_directive_error`) via shell commands or Python callbacks.
- **task-68** тАФ Output format adapters: plugin interface for custom formats beyond markdown/HTML. `perseus render --format json` returns structured `{resolved, directives}` output.
- **task-69** тАФ Foreign resolver protocol: `@perseus <url>` fetches rendered context from remote Perseus serve instances. HMAC signature verification, TTL caching, graceful degradation.
- **task-70** тАФ Custom schema validators: plugin validators in `.perseus/schemas/`. Referenced via `schema="plugin:my-validator"`. Works alongside the built-in validator.
- **task-71** тАФ Pipe syntax: lightweight chaining тАФ `@query "ls" | @cache ttl=300`. Left-to-right resolution, output of stage N becomes input of stage N+1.
- **task-72** тАФ Event webhooks: POST render lifecycle events to external URLs with optional HMAC-SHA256 signing. Config-driven with per-event selection.
- **task-73** тАФ Tool directive integration: `@tool "path/to/tool"` with config-based allowlist, argument restrictions, timeouts, and output size caps.
- **task-74** тАФ Directive aliasing: config-driven shorthand тАФ `@qтЖТ@query`, `@svcтЖТ@services`. Single-pass expansion, built-ins always win collisions.

**Phase 25 тАФ MCP Deep Integration:**

- **task-75** тАФ Expose every directive as an MCP tool. `perseus mcp serve` runs a JSON-RPC 2.0 MCP server over stdio. Each `DIRECTIVE_REGISTRY` entry becomes a `perseus_<name>` tool with auto-generated descriptions and input schemas. Trust gates enforced per-tool. Backward compatible with existing `perseus_get_context` / `perseus_get_health`.

## [1.0.2] тАФ 2026-05-23

**Bug fixes (Opus 4.7 Max / Codex 5.5 Extreme High benchmarks):**

- **task-63** тАФ Fixed `Path.write_text` missing encoding on Windows тАФ emoji (ЁЯУМ) crash in default prompts.
- **task-64** тАФ Fixed `/bin/bash` unreachable on native Windows Python тАФ added `_get_shell()` helper
  using `shutil.which()` with system-default fallback for `@query`, `@services`, and `@agent`.
- **task-65** тАФ Fixed `@query` binary stdout NoneType crash тАФ guarded `result.stdout` with `or ""`.
- **task-66** тАФ Fixed `perseus --help` crash on Windows (Mn─Уm─У macron `─У` can't encode to `cp1252`) тАФ
  added `sys.stdout/stderr.reconfigure(encoding="utf-8")` at import time.

**New features:**

- **task-67** тАФ **`render.max_query_bytes`** (default 256 KB) тАФ caps runaway `@query` stdout with a
  visible truncation marker. Prevents 12 MB scanner output from silently inflating
  context documents (47├Ч output reduction demonstrated).
- **task-68** тАФ **Configurable `@query` timeout** тАФ `render.query_timeout_s` (default 30s) and per-directive
  `timeout=N` modifier (e.g. `@query "..." timeout=120`).
- **task-69** тАФ **`render.parallel_services`** (opt-in, default off) тАФ concurrent `@services` health checks
  via `ThreadPoolExecutor`. 100 services go from ~5 min serial to ~3 s parallel.
- **task-70** тАФ **`render.parallel_queries`** (opt-in, default off) тАФ pre-scans top-level `@query` directives
  and resolves them concurrently. Directives inside `@if` branches remain sequential.

**Integrations:**

- **task-71** тАФ **VS Code / Cursor extension** тАФ auto-renders on `.perseus/context.md` save, status bar
  indicator, auto-detects target assistant file, watch mode.
- **task-72** тАФ **Claude Code session hook** тАФ one `curl` install, runs `perseus render` before every
  Claude Code session.
- **task-73** тАФ **GitHub Action** тАФ renders context on push/schedule, commits back to repo so every
  developer gets pre-resolved context without installing Perseus locally.

**Multi-agent coordination:**

- **task-74** тАФ **Shared checkpoint store** тАФ agents across machines/sessions share a single checkpoint
  store (config: `checkpoints.store` path, accessible via NFS/SMB/unison).
- **task-75** тАФ **Lock file mechanism** тАФ `os.O_CREAT | os.O_EXCL` atomic lock in the checkpoint
  store prevents concurrent writers from clobbering. Retries with backoff for ~11s before failing
  gracefully. NFS-safe (O_CREAT | O_EXCL is atomic cross-filesystem).
- **task-76** тАФ **Checkpoint recovery** тАФ `perseus recover --from <store>` reads the latest checkpoint
  and prints the workspace/task/status triplet so an agent dropped into a terminal knows exactly
  where to resume.

**Benchmarks:**

- Extreme scaling sweep on Linux: 10 тЖТ 10,000 `@query` directives, 4 modes each
  (sequential, cached, parallel, cached+parallel). Cache warm time stays flat at
  ~0.3тАУ0.5s regardless of scale. 10,000 queries at 0.52s warm (25├Ч vs 13.1s cold).
- Integrated heavy benchmark suite (`benchmark/heavy/`) with 4 reports, setup harnesses,
  machine-readable result JSONs from Claude Code Opus 4.7 and Codex 5.5 Extreme High runs.
- Efficiency infographic on README showing coldтЖТwarm scaling curve and 40├Ч warm speedup.
