Sibyl gives the agent perfect recall. But recall isn't the whole problem. The agent still has to ask: "where am I, what machine is this, what branch, what conventions apply, what were we working on?" Those are orientation questions — and they burn turns whether retrieval is perfect or not.

What This Benchmark Measures

Given perfect memory retrieval (Sibyl's 350/350), how many turns does the agent waste on orientation before it starts doing real work?

We seeded a Sibyl Memory database with a realistic project corpus — 70 entities across 9 categories, 292 KB SQLite — simulating months of accumulated project knowledge:

What Those Discovery Turns Look Like

For task #1 ("Fix credential redaction bug"), the agent's first 8 turns with Sibyl only:

Turn 1: sibyl_search("credential redaction") → 1 hit, 84 tokens Turn 2: sibyl_recall("auth", "github-token-extraction") → exact match Turn 3: sibyl_search("renderer component") → 8 hits, 310 tokens Turn 4: sibyl_recall("project", "perseus") → project context Turn 5: sibyl_recall("convention", "fix-root-cause") → workflow rule Turn 6: terminal: git branch --show-current → main Turn 7: terminal: ls src/perseus/renderer.py → file exists Turn 8: sibyl_recall("convention", "perseus-ci-rebuild") → build process Turn 9: [actual work begins]

With Perseus: All of the above is in AGENTS.md before turn 1. Agent reads context and starts working immediately.

Reproducibility

All data is publicly available. The benchmark corpus, task suite, and Perseus context template are in the Perseus repository under bench/.

pip install perseus-ctx sibyl-memory-client export SIBYL_MEMORY_ENABLED=1 PERSEUS_ALLOW_DANGEROUS=1 perseus render bench/bench_context.md --output AGENTS.md # Agent starts with full orientation. # Compare session length with and without AGENTS.md for any task.

What This Does Not Prove

  • Sibyl's benchmarks are independently verified by external testers. This benchmark uses a self-seeded corpus. An independent tester running the same methodology on their own project would strengthen the result.
  • Model variance matters. Different LLMs may burn different numbers of discovery turns. Sibyl's benchmarks control for this by testing multiple models (Sonnet, Opus); we haven't done multi-model runs yet.
  • Corpus size matters. 70 entities is a real project, not a lab experiment — but a 500-entity corpus (matching Sibyl's 500-company scale) would amplify the savings.
  • Task difficulty varies. Simple bug fixes need fewer discovery turns than architecture work. The 10-task suite covers both.

Bottom Line

Sibyl Memory gives the agent perfect recall. Perseus makes sure the agent never wastes a turn asking "where am I and what am I doing?"

Together they answer both questions that matter at session start: what do we know (Sibyl's 350/350 retrieval) and what's happening now (Perseus's 0-turn orientation). The result is an agent that is productive from turn 1.
