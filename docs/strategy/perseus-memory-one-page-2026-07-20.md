# Perseus one-page memory strategy

Date: July 20, 2026

Atlassian appears to be converging on a managed, product-native enterprise memory platform across Rovo, Jira, Confluence, chat, Teamwork Graph, and agent sessions.

Perseus should not compete with that directly.

## Position
Perseus and Perseus Vault should be the durable cognition layer for operators and agents.

## Atlassian likely owns
- product-native relevance
- permission-aware retrieval
- live work-graph context
- tenant-managed memory behavior

## Perseus and Vault should own
- long-horizon continuity
- explicit user control
- transparent why-am-I-seeing-this explanations
- correction and supersession
- portable synthesis across tools and sessions

## Now
- define served-memory views
- define retrieval explanation metadata
- formalize Vault taxonomy and precedence
- sharpen source anchors and correction workflows
- publish non-competitive positioning

## Next
- use served memory for briefing outputs
- add scope-aware serving controls
- add retention and decay controls
- design served-memory APIs for synthesis

## Rule
Do not build "Atlassian memory, but better."
Build the durable cognition layer that remains valuable when Atlassian memory exists.
