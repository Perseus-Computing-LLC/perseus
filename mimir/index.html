✦ Mimir · Persistent memory

Memory that survives the session.

Mimir is a single-file, fully-local memory server for AI agents — SQLite + FTS5, AES-256-GCM encrypted, 40 MCP tools, zero cloud dependencies. It's the default memory engine for Perseus, and it works standalone with any MCP client.

✦ Bring your own memory

One database. Every agent, every machine.

AI memories are historically locked inside one agent framework. Mimir decouples semantic memory into a portable file — point every agent, on every machine, at the same mimir.db. ACID-compliant and multi-writer safe.

⬡

Encrypted at rest

AES-256-GCM on all persistent memory. Self-hosted — data never leaves your infrastructure. Air-gap ready.

⟲

Hybrid recall

FTS5 + vector search with Porter stemming. Sub-millisecond recall, cold or warm. No external embedding API required.

◷

Confidence decay

Ebbinghaus-style half-lives: Insights 30d, Architecture 90d, Decisions 180d. Stale facts fade unless reinforced.

⇄

Multi-device sync

It's just a file — sync it with Syncthing, Nextcloud, or Dropbox. Federated multi-machine memory with Ed25519 signing.

◇

40 MCP tools

Remember, recall, link, traverse, journal, synthesize, decay, vault export/import — a full memory surface, MCP-native.

⊞

Three deployment modes

Standalone MCP server, the default Perseus backend, or a memory backend for other frameworks. MIT licensed.

✦ How it works

Write once. Recall forever.

Mimir runs as an MCP server next to your agent. Agents write structured entities — category, key, confidence — and recall them by full-text or vector similarity across any future session.
