§ 01

Bring Your Own Memory (BYOM)

Mimir introduces the Bring Your Own Memory (BYOM) architectural design pattern. Instead of locking agent state, context history, and learned preferences inside proprietary developer SDKs or isolated sessions, Mimir decouples semantic memory into a lightweight, portable database.

Decoupled state

Standalone sidecar

Point local and remote runtimes at the identical SQLite backend. No matter what container, host machine, or agent workspace is running, they share one cohesive recollection.

Infinite portability

Single-file SQLite

All of your memories are stored in a simple, compact mimir.db database file. Back it up, sync it via Syncthing or Dropbox, or host it on your local NAS for easy device-wide access.

Hybrid resolution

Integrated with Perseus

Mimir works natively with the Perseus context engine, pre-resolving long-term learned experiences, workspace structures, and developer choices directly into the system prompt.

Zero lock-in

Open-source middleware

Because Mimir runs as an independent sidecar communicating over MCP, you have full control over your data. No proprietary cloud interfaces, and no vendor lock-in.

§ 02

Extreme Performance

Most semantic memory solutions drag in complex vector indexing libraries, PyTorch model footprints, and slow, expensive cloud embedding APIs. Mimir proves that for developer context, keyword search combined with BM25 ranking is 100x faster, incredibly reliable, and free.

P50 Query Latency
0.09ms
Cold-start database query latency. Returns high-relevance matches instantly.
Warm Query Latency
0.12ms
Consistent warm query P50 latency. Remains flat even as memory scale increases.
System RAM Footprint
< 15MB
Ultra-lightweight Rust daemon. Leaves resources completely free for your models and builds.
SQLite FTS5

Porter Stemming Search

SQLite FTS5 full-text indexing with porter stemming. Words like testing, tested, and tests map to the identical stem, ensuring high-relevance keyword search matches.

Smart query fallback

Multi-word LIKE optimization

Standard SQLite FTS5 MATCH treats multi-word strings as exact phrases. Mimir automatically sanitizes input and falls back to SQL LIKE operators to find partial matches across multiple terms.

§ 03

Model Context Protocol Compliance

As a fully compliant Model Context Protocol server, Mimir registers semantic memory reading and writing directly as system tools. Any MCP-enabled coding assistant can automatically query, explore, or store project-level learnings.

Recall tool

mimir_recall

Query the semantic memory vault. Performs FTS5 BM25 search with fallback sanitization to retrieve relevant contextual fragments instantly.

Store tool

mimir_store

Save a new learning. Accepts markdown content, semantic tags, a classification type (e.g., insight, convention), and an importance rating.

Stats tool

mimir_stats

Fetches vault statistics. Inspect database sizing, record counts, and categorical breakdowns directly from your assistant terminal.

Health tool

mimir_health

Ensures structural integrity. Verifies database read/write capabilities, index files, and daemon connection status.

§ 04

Device & Team Synchronization

Mimir operates smoothly on everything from isolated developer workstations to distributed, multi-machine dev stacks. Point multiple servers, local agents, and CI pipelines to a single shared file sync folder to establish immediate semantic alignment.

Syncthing / Dropbox Sync Strategy

Point your Mimir database path to a folder synced by Syncthing, Nextcloud, Resilio, or Dropbox. SQLite's ACID-compliant architecture combined with file locking allows secure, multi-machine memory propagation:

Workstation A (Save Insight) → mimir_store → synced/mimir.db ← (Sync Engine) → Workstation B (Retrieve context)
§ 05

Quickstart Setup

Install Mimir
# 1. Run the one-shot bootstrap installation
curl -sSL https://raw.githubusercontent.com/tcconnally/mimir/main/scripts/bootstrap.sh | bash

# 2. Build from source if preferred
git clone https://github.com/tcconnally/mimir.git ~/.mimir
cd ~/.mimir && cargo build --release
cp target/release/mimir ~/.local/bin/mimir
Configure Perseus Integration
# In your workspace .perseus/config.yaml:
mimir:
  enabled: true
  transport: "stdio"
  command:
    - "/usr/local/bin/mimir"
    - "--db"
    - "~/.mimir/data/mimir.db"
  timeout_s: 30.0
  merge_strategy: "local_first"
  fallback_to_local: true
  circuit_breaker:
    threshold: 3
    cooldown: 120
  retry_policy:
    max_attempts: 3
    backoff_base: 1.5
