Why it matters

The value isn't only cheaper tokens. It's deployments you otherwise couldn't make.

A cloud-dependent memory layer is a non-starter the moment data can't leave the boundary. Perseus is local-first by construction: a single binary and a Python package that resolve context and store memory on your own disk.

That turns "we can't use agents here" into a supported configuration: on-premises, disconnected, or classified.

Security posture

Local-first, encrypted, inspectable.

Air-gapped

Fully offline by default. Zero cloud dependencies, no external network requests unless you explicitly configure one.

Encrypted at rest

Perseus Vault encrypts entity bodies with AES-256-GCM (opt-in via key file). The search index stays plaintext so recall still works.

Immutable audit

Every memory event is appended to a SHA-256 chained journal with actor attribution: a tamper-evident chain of custody.

Zero telemetry

No data collection, no usage stats, no phoning home, on either product. Nothing to disclose on a data-flow diagram.

MIT licensed

No copyleft, no GPL/AGPL, no vendor lock-in. Read the source, fork it, run it on your own hardware indefinitely.

Open formats

Memory exports to plain Markdown; context renders to plain files. Open licensing and open formats: your data is never trapped.

Deployment models

Three ways to run it inside the boundary.

Air-gapped

Runs with no network at all. Offline embeddings are compiled into the binary, so semantic recall needs no model download and no API key.

Security surface, per product

Hardened where it counts.

Perseus · context engine

  • →Read-only pre-processor: renders context, never stores it
  • →SSRF protection: URL allowlist, private-IP blocking, HMAC signature verification
  • →Workspace boundary enforcement: symlink escapes blocked
  • →Shell-executing directives gated behind explicit opt-in
  • →MCP SSE transport requires a bearer token

Perseus Vault · memory

  • →AES-256-GCM encryption of entity bodies at rest (opt-in key file)
  • →Immutable journal with actor attribution and SHA-256 chaining
  • →Bearer-token auth on SSE / HTTP transports (constant-time compare)
  • →One local SQLite file you own: no server, no cloud
  • →You control retention: soft-delete, archive, or permanent purge
